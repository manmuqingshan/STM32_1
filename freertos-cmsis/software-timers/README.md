# STM32 FreeRTOS Software Timers (Periodic & One-Shot)

This project demonstrates how to use software timers in FreeRTOS on STM32 using CMSIS-OS v2.  
We create both periodic and one-shot timers, execute callback functions through the timer daemon task, and control timer behavior dynamically using a button and LED.

## 📋 Overview

Software timers allow you to execute time-based functions without blocking tasks or wasting CPU cycles. Instead of using delay loops, FreeRTOS manages timers internally using a dedicated timer daemon task.

This project covers:

- **Periodic Timer** — Executes a callback every fixed interval (1 second)
- **One-Shot Timer** — Executes once after a delay (5 seconds)
- **Timer Reset Behavior** — Restarting a running one-shot timer resets its countdown
- **Dynamic Control** — Stopping and restarting timers based on button input

## 🛠 Hardware

- STM32L496 Nucleo board (any STM32 Nucleo board works with minor changes)
- USB cable for Virtual COM port (ST-LINK)
- Push button connected between PA3 and GND
- LED on PB7

| Timer | Type | Callback | Purpose |
|------|------|----------|--------|
| PeriodicTimer | Periodic | `PTCallback` | Prints message every 1 second |
| OneShotTimer | One-shot | `OSTCallback` | Turns off LED after 5 seconds |

## 💻 Code

### Timer Creation (Auto-generated by CubeMX)
```c
PeriodicTimerHandle = osTimerNew(PTCallback, osTimerPeriodic, NULL, &PeriodicTimer_attributes);
OneShotTimerHandle  = osTimerNew(OSTCallback, osTimerOnce,    NULL, &OneShotTimer_attributes);
````

---

### Start Periodic Timer

```c
void StartDefaultTask(void *argument)
{
    osTimerStart(PeriodicTimerHandle, 1000);

    for(;;)
    {
        osDelay(1);
    }
}
```

---

### Periodic Timer Callback

```c
void PTCallback(void *argument)
{
    printf("Data Sent from Periodic Timer's Callback\n");
}
```

---

### Button Handling + One-Shot Timer

```c
void StartDefaultTask(void *argument)
{
    osTimerStart(PeriodicTimerHandle, 1000);

    for(;;)
    {
        if (HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_3) == GPIO_PIN_RESET)
        {
            HAL_GPIO_WritePin(GPIOB, GPIO_PIN_7, GPIO_PIN_SET);

            osTimerStart(OneShotTimerHandle, 5000);

            if (osTimerIsRunning(PeriodicTimerHandle) == 1)
            {
                osTimerStop(PeriodicTimerHandle);
            }
        }
        osDelay(1);
    }
}
```

---

### One-Shot Timer Callback

```c
void OSTCallback(void *argument)
{
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_7, GPIO_PIN_RESET);

    printf("##### TIMER EXPIRED #####\n\n");

    if (osTimerIsRunning(PeriodicTimerHandle) == 0)
    {
        osTimerStart(PeriodicTimerHandle, 1000);
    }
}
```

## 🔁 Key Behavior

* Periodic timer runs continuously every 1 second
* Button press:

  * Turns LED ON
  * Starts one-shot timer (5 seconds)
  * Stops periodic timer
* If button is pressed again before 5 seconds:

  * One-shot timer resets (extends delay)
* When one-shot expires:

  * LED turns OFF
  * Periodic timer restarts

---

## ⚠️ Important Notes

* Timer callbacks run inside the **timer daemon task**
* Keep callbacks short and non-blocking
* Avoid using `osDelay()` or blocking APIs inside callbacks
* Calling `osTimerStart()` on a running timer resets its countdown
