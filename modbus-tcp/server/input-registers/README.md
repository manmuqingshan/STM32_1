# STM32 Modbus TCP Server — Input Registers & PWM
 
Part 4 of the STM32 Modbus TCP Server series. This project implements Modbus TCP input register support on the STM32H755 Nucleo board using the Mongoose networking library. A potentiometer connected to ADC1 provides live 16-bit values that a Modbus client can read using function code 4. The first holding register is connected to Timer 2 PWM output to control LED brightness.

## Hardware Used
 
| Component | Details |
|-----------|---------|
| Board | STM32H755 Nucleo |
| Potentiometer | Connected to PC0 — ADC1 Channel 10 |
| LED | Connected to PA3 — Timer 2 Channel 4 (PWM) |
| Power | 3.3V from Nucleo board |
 
---
 
## What This Project Does
 
- Reads a live 16-bit ADC value from a potentiometer on PC0
- Exposes the ADC value as Modbus input register at address 30001 (FC4)
- Serves 5 additional input registers from a fixed database (addresses 30002–30006)
- Controls LED brightness via PWM by writing to holding register 40001 (FC6 / FC16)
- Serves a live Web UI dashboard via Mongoose showing both input and holding registers
- Updates the Web UI every 200 ms via WebSocket
 
---
 
## Function Codes Handled
 
| FC | Hex | Operation |
|----|-----|-----------|
| 3 | 0x03 | Read holding registers |
| 4 | 0x04 | Read input registers |
| 6 | 0x06 | Write single holding register |
| 16 | 0x10 | Write multiple holding registers |

## CubeMX Configuration
 
| Peripheral | Pin | Mode |
|------------|-----|------|
| ADC1 Channel 10 | PC0 | Single-ended, 16-bit |
| TIM2 Channel 4 | PA3 | PWM Generation, Period = 65535 |
| ETH | — | RMII mode |
| USART3 | — | Async (for serial log) |
 
**ADC clock note:** Set the ADC clock source to Peripheral Clock to avoid exceeding the ADC clock limit. This can be changed under Clock Configuration in CubeMX.

## How to Build
 
1. Clone this repository
2. Open the `.ioc` file in STM32CubeMX and generate code
3. If you get ADC-related build errors, copy the ADC HAL source files from `Drivers/STM32H7xx_HAL_Driver/Src` into your project's HAL Drivers `Src` folder
4. Build and flash using STM32CubeIDE
 
---
 
## Testing
 
**Web UI:** Open the board's IP address in a browser. The dashboard shows all 12 holding registers and 6 input registers. Turn the potentiometer and watch register 30001 update in real time. Type a value into holding register 40001 to change the LED brightness.
 
**Modbus Client (e.g. Simply Modbus TCP):**
- Read input registers: start address 30001, count 6, FC4
- Write holding register: address 40001, value 0–65535, FC6
