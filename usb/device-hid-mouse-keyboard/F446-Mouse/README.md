# STM32 USB HID — Mouse

Part 3 (Mouse) of the STM32 USB series. This project configures an STM32 as a USB HID mouse using STM32CubeMX and HAL. The default mouse descriptor generated by CubeMX is used as-is. An analog joystick is read over ADC with DMA for cursor movement, and three push buttons handle left click, right click, and middle click. No custom drivers are needed on the PC side — Windows and macOS recognise the device automatically as a standard mouse.

## What This Project Does

- Configures USB OTG FS in Device Only mode using STM32CubeMX
- Uses the default mouse HID report descriptor generated by CubeMX — no descriptor changes needed
- Reads X and Y axis values from an analog joystick over ADC with DMA in circular mode
- Converts absolute ADC values (0–255) into relative movement deltas (±5 pixels) using threshold checks
- Reads three push buttons for left click, right click, and middle click
- Sends a 4-byte HID report (buttons, X delta, Y delta, scroll) to the host every 10 ms

---

## HID Report Structure

The default mouse descriptor generated by CubeMX sends 4 bytes per report:

| Byte | Content | Notes |
|------|---------|-------|
| 0 | Button states | Bit 0 = left click, Bit 1 = right click, Bit 2 = middle click |
| 1 | X movement | Signed, relative delta (-127 to +127) |
| 2 | Y movement | Signed, relative delta (-127 to +127) |
| 3 | Scroll wheel | Signed, relative delta — sent as 0 (no wheel) |

No changes are needed in `usbd_hid.c` or `usbd_hid.h` for the mouse project. The descriptor and endpoint size (`0x04`) are correct by default.

---

## Key Concept: Absolute vs Relative Data

The gamepad (Part 2) sent **absolute** ADC values directly to the HID buffer. The mouse descriptor expects **relative** data — the value represents how many pixels the cursor should move, not where it is.

The ADC gives a value from 0 to 255. At rest, the joystick sits near the midpoint (~128). We apply threshold checks to convert this into a movement delta:

```c
void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef *hadc)
{
    if      (ADC_Val[0] > 200) HID_Buffer[1] =  5;   // Tilt right → move right
    else if (ADC_Val[0] <  55) HID_Buffer[1] = -5;   // Tilt left  → move left
    else                       HID_Buffer[1] =  0;   // Centre     → no movement

    if      (ADC_Val[1] > 200) HID_Buffer[2] =  5;   // Tilt down  → move down
    else if (ADC_Val[1] <  55) HID_Buffer[2] = -5;   // Tilt up    → move up
    else                       HID_Buffer[2] =  0;   // Centre     → no movement
}
```

Adjust the thresholds (55 and 200) to match your joystick's dead zone if the cursor drifts at rest.

## Complete main.c User Code

```c
/* USER CODE BEGIN 0 */
extern USBD_HandleTypeDef hUsbDeviceFS;
int8_t HID_Buffer[4] = {0};
uint8_t ADC_Val[2];

void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef *hadc)
{
    if      (ADC_Val[0] > 200) HID_Buffer[1] =  5;
    else if (ADC_Val[0] <  55) HID_Buffer[1] = -5;
    else                       HID_Buffer[1] =  0;

    if      (ADC_Val[1] > 200) HID_Buffer[2] =  5;
    else if (ADC_Val[1] <  55) HID_Buffer[2] = -5;
    else                       HID_Buffer[2] =  0;
}
/* USER CODE END 0 */

/* USER CODE BEGIN 2 */
HAL_ADC_Start_DMA(&hadc1, (uint32_t *)ADC_Val, 2);
/* USER CODE END 2 */

/* USER CODE BEGIN 3 */
if (HAL_GPIO_ReadPin(L_BUTTON_GPIO_Port, L_BUTTON_Pin) == 0)
    HID_Buffer[0] = 0x01;  // Bit 0: left click
if (HAL_GPIO_ReadPin(R_BUTTON_GPIO_Port, R_BUTTON_Pin) == 0)
    HID_Buffer[0] = 0x02;  // Bit 1: right click
if (HAL_GPIO_ReadPin(M_BUTTON_GPIO_Port, M_BUTTON_Pin) == 0)
    HID_Buffer[0] = 0x04;  // Bit 2: middle click

USBD_HID_SendReport(&hUsbDeviceFS, (uint8_t *)HID_Buffer, 4);
HID_Buffer[0] = 0;         // Reset button state after sending
HAL_Delay(10);
/* USER CODE END 3 */
```

> **Note:** `HID_Buffer` is declared as `int8_t` to hold negative movement values, but `USBD_HID_SendReport` expects `uint8_t *`. The typecast `(uint8_t *)HID_Buffer` resolves this — the underlying memory is the same.

## Testing

**Windows:** Open Device Manager → Human Interface Devices. The device should appear as **STM32 Mouse**. Move the joystick — the system cursor should respond.

**macOS:** The cursor should move immediately after connecting. Check Settings → General → System Report → USB to confirm enumeration.

**Buttons:** Test left click, right click, and middle click by pressing each button.
